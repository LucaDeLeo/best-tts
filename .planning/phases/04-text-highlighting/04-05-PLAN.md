Update the TTS generation flow to use highlighting chunks.

1. Add new handler for TTS_GENERATE that initializes highlighting first:

```typescript
// In handleServiceWorkerMessage, add case for new highlighting-aware flow:
case MessageType.TTS_GENERATE: {
  const { text, voice } = message as TTSGenerateMessage;

  // Get active tab
  const [tab] = await chrome.tabs.query({ active: true, currentWindow: true });
  if (!tab?.id) {
    sendResponse({ success: false, error: 'No active tab' });
    break;
  }

  // Determine mode based on extraction source
  // If pendingExtraction exists and is article, use overlay mode
  const { pendingExtraction } = await chrome.storage.session.get('pendingExtraction');
  const mode = pendingExtraction?.source === 'article' ? 'overlay' : 'selection';

  // Initialize highlighting in content script
  // This returns the chunks that are aligned with DOM spans
  const highlightResult = await chrome.tabs.sendMessage(tab.id, {
    target: 'content-script',
    type: MessageType.INIT_HIGHLIGHTING,
    mode,
    text,
    title: pendingExtraction?.title
  });

  if (!highlightResult.success || !highlightResult.chunks) {
    // Fallback: proceed without highlighting
    console.warn('Highlighting init failed, proceeding without:', highlightResult.error);
    // Use existing TTS_GENERATE flow via offscreen
    routeToOffscreen(message, sendResponse);
    break;
  }

  // Use chunks from highlighting (guaranteed alignment)
  const chunks = highlightResult.chunks;
  const token = generateToken();

  updatePlaybackState({
    status: 'generating',
    generationToken: token,
    chunks,
    totalChunks: chunks.length,
    currentChunkIndex: 0,
    activeTabId: tab.id
  });

  // Start playing first chunk
  playChunk(0).catch(console.error);

  sendResponse({
    success: true,
    generationToken: token,
    chunks
  });
  break;
}
```

Note: This replaces the forwardTo: 'offscreen' pattern for TTS_GENERATE. The offscreen document is still used for chunk generation, but chunking now happens in content script.

2. Update the offscreen routing to only handle TTS_GENERATE_CHUNK:

The existing flow routes TTS_GENERATE to offscreen which returns chunks. We need to change this:
- Content script now provides chunks via INIT_HIGHLIGHTING
- Service worker stores chunks and calls playChunk()
- playChunk() calls offscreen for TTS_GENERATE_CHUNK (single chunk generation)

This keeps the TTS engine in offscreen but moves chunking to content script for highlighting alignment.
