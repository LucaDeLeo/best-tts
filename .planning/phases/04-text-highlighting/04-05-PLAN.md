Update the TTS generation flow to use highlighting chunks.

**MESSAGE ROUTING CHANGES:**

The service worker currently routes TTS_GENERATE to offscreen which does chunking. After this change:
- `TTS_GENERATE`: Handled IN SERVICE WORKER (not forwarded to offscreen) - initializes highlighting in content script, gets chunks back
- `TTS_GENERATE_CHUNK`: STILL forwarded to offscreen for audio generation (unchanged)
- `PLAY_AUDIO`: Sent to content script with chunkIndex (unchanged)

1. Update the message routing switch statement to handle TTS_GENERATE directly:

```typescript
// BEFORE: TTS_GENERATE was forwarded to offscreen
// case MessageType.TTS_GENERATE:
//   forwardToOffscreen(message, sendResponse);
//   break;

// AFTER: Handle TTS_GENERATE in service worker to integrate highlighting
case MessageType.TTS_GENERATE: {
  const { text, voice } = message as TTSGenerateMessage;

  // Get active tab
  const [tab] = await chrome.tabs.query({ active: true, currentWindow: true });
  if (!tab?.id) {
    sendResponse({ success: false, error: 'No active tab' });
    break;
  }

  // Determine mode based on extraction source
  // If pendingExtraction exists and is article, use overlay mode
  const { pendingExtraction } = await chrome.storage.session.get('pendingExtraction');
  const mode = pendingExtraction?.source === 'article' ? 'overlay' : 'selection';

  // Initialize highlighting in content script
  // This returns the chunks that are aligned with DOM spans
  const highlightResult = await chrome.tabs.sendMessage(tab.id, {
    target: 'content-script',
    type: MessageType.INIT_HIGHLIGHTING,
    mode,
    text,
    title: pendingExtraction?.title
  });

  if (!highlightResult.success || !highlightResult.chunks) {
    // Fallback: proceed without highlighting
    console.warn('Highlighting init failed, proceeding without:', highlightResult.error);
    // Fallback: do chunking in service worker and proceed without DOM alignment
    const fallbackChunks = chunkTextForTTS(text); // existing chunking function
    startPlaybackWithChunks(fallbackChunks, voice, tab.id, sendResponse);
    break;
  }

  // Use chunks from highlighting (guaranteed DOM alignment)
  startPlaybackWithChunks(highlightResult.chunks, voice, tab.id, sendResponse);
  break;
}

// TTS_GENERATE_CHUNK is STILL routed to offscreen (audio generation per chunk)
case MessageType.TTS_GENERATE_CHUNK:
  forwardToOffscreen(message, sendResponse);
  break;
```

2. Add helper function for starting playback:

```typescript
async function startPlaybackWithChunks(
  chunks: string[],
  voice: string,
  tabId: number,
  sendResponse: (response: any) => void
): Promise<void> {
  const token = generateToken();

  updatePlaybackState({
    status: 'generating',
    generationToken: token,
    chunks,
    totalChunks: chunks.length,
    currentChunkIndex: 0,
    activeTabId: tabId
  });

  // Start playing first chunk (this calls offscreen for TTS_GENERATE_CHUNK)
  playChunk(0).catch(console.error);

  sendResponse({
    success: true,
    generationToken: token,
    chunks
  });
}
```

**Summary of routing:**
- TTS_GENERATE: Service worker handles, sends INIT_HIGHLIGHTING to content script, then uses returned chunks
- TTS_GENERATE_CHUNK: Forward to offscreen (generates audio for single chunk)
- PLAY_AUDIO: Send to content script with chunkIndex (triggers highlight)
