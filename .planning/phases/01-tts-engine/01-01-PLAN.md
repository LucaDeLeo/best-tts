  <action>
Create a new npm project with the following structure:

1. Initialize package.json with:
   - name: "best-tts"
   - type: "module"
   - scripts: { "dev": "vite", "build": "vite build" }

2. Install dependencies:
   - Runtime: kokoro-js, idb-keyval, onnxruntime-web, @huggingface/transformers
   - Dev: @crxjs/vite-plugin, vite, typescript, @types/chrome, vite-plugin-static-copy

   NOTE: onnxruntime-web is explicitly listed even though kokoro-js depends on it, to ensure
   it's available at the top level for WASM file copying and explicit import in tts-engine.ts.

   NOTE: @huggingface/transformers is listed even though kokoro-js depends on it, because we
   need to import its `env` object to configure IndexedDB caching for offline support. Without
   explicit cache configuration, transformers.js may fail to cache models in extension context.

3. Create tsconfig.json:
   - target: ESNext
   - module: ESNext
   - moduleResolution: bundler
   - strict: true
   - include: ["src"]
   - types: ["chrome"]

4. Create vite.config.ts:
   - Import crx from @crxjs/vite-plugin
   - Import manifest from src/manifest.json
   - Configure crx plugin with manifest
   - Set build.target to "esnext"
   - Set build.minify to false (easier debugging)
   - Add offscreen document to rollupOptions.input (see step 6)

5. Create src/manifest.json (MV3):
   - manifest_version: 3
   - name: "Best TTS"
   - version: "0.1.0"
   - description: "High-quality local TTS"
   - permissions: ["offscreen", "storage"]
   - host_permissions: ["https://huggingface.co/*", "https://*.huggingface.co/*", "https://cdn-lfs.huggingface.co/*", "https://cdn-lfs-us-1.huggingface.co/*", "https://cdn.jsdelivr.net/*"]

   NOTE: Hugging Face uses multiple CDN domains for LFS (Large File Storage) where model files
   are hosted. The cdn-lfs*.huggingface.co patterns cover regional CDN endpoints used for
   model downloads. Without these, offline caching will fail on first download.
   - background: { service_worker: "background/service-worker.ts", type: "module" }
   - action: { default_popup: "popup/index.html" }
   - content_security_policy: { extension_pages: "script-src 'self' 'wasm-unsafe-eval'; object-src 'self';" }

IMPORTANT: The CSP must include 'wasm-unsafe-eval' for ONNX Runtime WASM to work.

6. Update vite.config.ts to include the offscreen document as an additional HTML entry:

   CRXJS does NOT automatically include the offscreen document in the build because it's not
   referenced in manifest.json (Chrome's offscreen API uses runtime paths, not manifest entries).

   Add to vite.config.ts build.rollupOptions.input:
   ```typescript
   build: {
     target: 'esnext',
     minify: false,
     rollupOptions: {
       input: {
         offscreen: 'src/offscreen/offscreen.html'
       }
     }
   }
   ```

   This ensures the offscreen document is bundled into dist/ so chrome.offscreen.createDocument
   can find it at runtime.

  <action>
Create src/lib/messages.ts with TypeScript types and constants for all extension message passing:

```typescript
// Message targets
export type MessageTarget = 'offscreen' | 'service-worker' | 'popup';

// Message types enum
export const MessageType = {
  // TTS lifecycle
  TTS_INIT: 'tts-init',
  TTS_GENERATE: 'tts-generate',
  TTS_STOP: 'tts-stop',
  TTS_LIST_VOICES: 'tts-list-voices',

  // Progress updates
  DOWNLOAD_PROGRESS: 'download-progress',
  GENERATION_COMPLETE: 'generation-complete',

  // Status
  GET_STATUS: 'get-status',
  STATUS_UPDATE: 'status-update',
} as const;

export type MessageTypeValue = typeof MessageType[keyof typeof MessageType];

// Base message structure
interface BaseMessage {
  target: MessageTarget;
  type: MessageTypeValue;
}

// Specific message types
export interface TTSInitMessage extends BaseMessage {
  type: typeof MessageType.TTS_INIT;
}

export interface TTSGenerateMessage extends BaseMessage {
  type: typeof MessageType.TTS_GENERATE;
  text: string;
  voice: string;
}

export interface TTSStopMessage extends BaseMessage {
  type: typeof MessageType.TTS_STOP;
}

export interface TTSListVoicesMessage extends BaseMessage {
  type: typeof MessageType.TTS_LIST_VOICES;
}

export interface DownloadProgressMessage extends BaseMessage {
  type: typeof MessageType.DOWNLOAD_PROGRESS;
  progress: {
    file: string;
    loaded: number;
    total: number;
    percent: number;
  };
}

// Union type for all messages
export type TTSMessage =
  | TTSInitMessage
  | TTSGenerateMessage
  | TTSStopMessage
  | TTSListVoicesMessage
  | DownloadProgressMessage;

// Response types
export interface TTSResponse {
  success: boolean;
  error?: string;
}

export interface VoiceListResponse extends TTSResponse {
  voices?: string[];
}

export interface StatusResponse extends TTSResponse {
  initialized: boolean;
  currentVoice?: string;
  isPlaying?: boolean;
}

// Helper to create messages
export function createMessage<T extends TTSMessage>(
  target: MessageTarget,
  type: T['type'],
  payload?: Omit<T, 'target' | 'type'>
): T {
  return { target, type, ...payload } as T;
}
```

This provides compile-time safety for all message passing between extension components.
